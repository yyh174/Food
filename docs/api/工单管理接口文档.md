# 工单管理接口文档

## 通用接口规范

### 1) 统一返回结构

所有接口统一返回：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

- `code`：业务状态码，`200` 表示成功
- `message`：提示信息
- `data`：响应数据，列表接口通常返回分页对象，删除等操作可为 `null`

### 2) 错误码约定

| 错误码 | 含义 |
|------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未登录或登录已过期 |
| 403 | 无权限访问 |
| 404 | 资源不存在 |
| 409 | 业务冲突（如状态不允许、数据重复） |
| 500 | 服务器内部错误 |

### 3) 分页参数约定

- 请求参数统一使用：`page`（页码，从 1 开始）、`pageSize`（每页条数）
- 分页响应统一包含：`records`、`total`、`page`、`pageSize`

### 4) 时间格式约定

- 日期：`yyyy-MM-dd`
- 日期时间：`yyyy-MM-dd HH:mm:ss`
- 默认时区：`Asia/Shanghai`（东八区）

### 5) 鉴权方式约定

- 除登录/注册等白名单接口外，均需携带 JWT 访问令牌
- 请求头统一使用：`Authorization: Bearer <token>`

## 概述

- **模块说明**：工单管理
- **权限说明**：
  - 租户管理员（`tenant_admin`）：创建工单、派单、核销、归档、打回
  - 店长（`shop_owner`）：处理自己门店的工单、提交整改
- **数据范围**：店长只能操作自己关联门店的工单

## 工单状态流转图

### 租户管理员视角

```
┌────────────────────────────────────────────────────────────────────┐
│  1待处理(待派单) → 2处理中 → 3待核销 → 4已归档                   │
│                       ↑                    │                        │
│                       └──── 打回(回到2) ────┘                        │
│  5已忽略：可由租户管理员在特定场景直接标记，流程终态              │
└────────────────────────────────────────────────────────────────────┘
```

| 状态 | 说明 |
|------|------|
| 1 待处理(待派单) | 新建工单，等待派单 |
| 2 处理中 | 已分配给店长处理中 |
| 3 待核销 | 店长提交整改，等待核销 |
| 4 已归档 | 核销后归档，或直接关闭 |
| 5 已忽略 | 特定场景下可忽略，进入终态 |

### 店长视角

```
┌──────────────────────────────────────────────────────────────┐
│  待处理 → 处理中 → 提交整改                                  │
│              ↑                                              │
│              └─── 打回后回来                                 │
└──────────────────────────────────────────────────────────────┘
```

| 状态 | 说明 |
|------|------|
| 待处理 | 店长收到派单，等待处理 |
| 处理中 | 店长正在处理 |
| 提交整改 | 店长提交整改，等待核销（打回后回到"处理中"） |

## 1. 工单类型

### 请求信息

| 项目 | 内容 |
|------|------|
| 接口路径 | `GET /api/tickets/types` |
| 请求格式 | Query Params |

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| type | String | 否 | 类型：system-系统预置，custom-自定义，默认全部 |

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| code | Integer | 状态码 |
| message | String | 提示信息 |
| data | Array | 工单类型列表 |

#### data[] 工单类型对象

| 参数名 | 类型 | 说明 |
|--------|------|------|
| id | Long | 类型ID |
| name | String | 类型名称 |
| type | String | 类型：system-系统预置，custom-自定义 |
| isDefault | Boolean | 是否系统预置 |
| supportReview | Boolean | 是否支持关联评价（差评预警类型为true） |

### 响应示例

```json
{
  "code": 200,
  "message": "success",
  "data": [
    { "id": 1, "name": "差评预警", "type": "system", "isDefault": true, "supportReview": true },
    { "id": 2, "name": "投诉预警", "type": "system", "isDefault": true, "supportReview": false },
    { "id": 3, "name": "指标预警", "type": "system", "isDefault": true, "supportReview": false },
    { "id": 4, "name": "自定义", "type": "custom", "isDefault": false, "supportReview": false }
  ]
}
```

## 6. 创建工单

### 请求信息

| 项目 | 内容 |
|------|------|
| 接口路径 | `POST /api/tickets` |
| 请求格式 | JSON Body |

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| title | String | 是 | 工单标题（最多100字） |
| description | String | 是 | 工单描述（最多1000字） |
| typeId | Long | 是 | 工单类型ID |
| shopId | Long | 是 | 关联门店ID |
| deadline | String | 否 | 截止时间（格式：yyyy-MM-dd HH:mm:ss） |
| images | Array[String] | 否 | 图片附件URL列表（最多9张） |
| reviewId | Long | 条件必填 | 关联评价ID（当 typeId 为"差评预警"时必填） |

### 请求示例

```json
{
  "title": "差评预警：门店1收到差评",
  "description": "手动创建的差评预警工单",
  "typeId": 1,
  "shopId": 1,
  "deadline": "2024-01-16 18:00:00",
  "reviewId": 5
}
```

### 响应示例

```json
{
  "code": 200,
  "message": "创建成功",
  "data": {
    "id": 10,
    "title": "差评预警：门店1收到差评",
    "status": 1,
    "priority": "high",
    "deadline": "2024-01-16 18:00:00",
    "createdAt": "2024-01-15 10:00:00"
  }
}
```

### 业务说明

- 仅租户管理员可创建工单
- 优先级根据截止时间动态计算：
  - 剩余时间 ≤ 1天：高优先级
  - 剩余时间 ≤ 3天：中优先级
  - 剩余时间 > 3天：低优先级
- 创建后状态为 `1`（待处理/待派单），需要派单后才能处理
- **差评预警类型**：必须关联一个评价（`reviewId`）

## 差评预警生成规则

### 自动生成（定时任务）

- 每小时扫描未处理的新增差评（1-2星）
- 发现差评后自动创建工单，状态为 `1`（待处理/待派单）
- 工单标题格式：`差评预警：{门店名称}收到{星级}星差评`
- 截止时间默认为：创建时间 + 24小时

### 手动创建

- 租户管理员选择工单类型为"差评预警"时
- 必须选择一个关联的评价（`reviewId`）
- 关联的评价必须是该门店的未处理差评
- 已关联工单的评价不可再次关联
