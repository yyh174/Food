# 门店管理接口文档

## 通用接口规范

### 1) 统一返回结构

所有接口统一返回：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

- `code`：业务状态码，`200` 表示成功
- `message`：提示信息
- `data`：响应数据，列表接口通常返回分页对象，删除等操作可为 `null`

### 2) 错误码约定

| 错误码 | 含义 |
|------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未登录或登录已过期 |
| 403 | 无权限访问 |
| 404 | 资源不存在 |
| 409 | 业务冲突（如状态不允许、数据重复） |
| 500 | 服务器内部错误 |

### 3) 分页参数约定

- 请求参数统一使用：`page`（页码，从 1 开始）、`pageSize`（每页条数）
- 分页响应统一包含：`records`、`total`、`page`、`pageSize`

### 4) 时间格式约定

- 日期：`yyyy-MM-dd`
- 日期时间：`yyyy-MM-dd HH:mm:ss`
- 默认时区：`Asia/Shanghai`（东八区）

### 5) 鉴权方式约定

- 除登录/注册等白名单接口外，均需携带 JWT 访问令牌
- 请求头统一使用：`Authorization: Bearer <token>`

## 概述

- **模块说明**：门店管理（租户管理员使用）
- **权限说明**：仅 `tenant_admin`（租户管理员）可访问
- **数据范围**：租户管理员只能操作自己租户下的门店

## 1. 门店列表

### 请求信息

| 项目 | 内容 |
|------|------|
| 接口路径 | `GET /api/shops` |
| 请求格式 | Query Params |

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| page | Integer | 否 | 页码，默认 1 |
| pageSize | Integer | 否 | 每页条数，默认 10 |
| keyword | String | 否 | 搜索关键词（匹配门店名称、编码） |
| status | Integer | 否 | 状态筛选：0-已停业，1-营业中，2-休息中 |

| 参数名 | 类型 | 说明 |
|--------|------|------|
| code | Integer | 状态码 |
| message | String | 提示信息 |
| data | Object | 数据对象 |

#### data 结构

| 参数名 | 类型 | 说明 |
|--------|------|------|
| records | Array | 门店列表 |
| total | Long | 总条数 |
| page | Integer | 当前页码 |
| pageSize | Integer | 每页条数 |

#### records[] 门店对象

| 参数名 | 类型 | 说明 |
|--------|------|------|
| id | Long | 门店ID |
| shopCode | String | 门店编码 |
| shopName | String | 门店名称 |

## 3. 创建门店

### 请求信息

| 项目 | 内容 |
|------|------|
| 接口路径 | `POST /api/shops` |
| 请求格式 | JSON Body |

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| shopName | String | 是 | 门店名称 |
| province | String | 否 | 省份 |
| city | String | 否 | 城市 |
| district | String | 否 | 区县 |
| address | String | 否 | 详细地址 |
| managerName | String | 否 | 店长姓名 |
| managerPhone | String | 否 | 店长电话 |
| meituanId | String | 否 | 美团门店ID |
| eleId | String | 否 | 饿了么门店ID |

### 请求示例

```json
{
  "shopName": "门店1",
  "province": "广东省",
  "city": "深圳市",
  "district": "南山区",
  "address": "科技园路100号",
  "managerName": "张三",
  "managerPhone": "13800138000",
  "meituanId": "MT123456",
  "eleId": "ELE789"
}
```

### 响应示例

```json
{
  "code": 200,
  "message": "创建成功",
  "data": {
    "id": 1,
    "shopCode": "SHOP001",
    "shopName": "门店1",
    "status": 1,
    "createdAt": "2024-01-15 10:30:00"
  }
}
```

### 业务说明

- `shopCode` 由系统自动生成，格式：`SHOP` + 6位序号（如 SHOP001）
- 新建门店默认状态为 `1`（营业中）
- `meituanId` 和 `eleId` 至少填写一个（评价来源于此）

## 4. 编辑门店

### 请求信息

| 项目 | 内容 |
|------|------|
| 接口路径 | `PUT /api/shops/{id}` |
| 请求格式 | Path Variable + JSON Body |

### 路径参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| id | Long | 是 | 门店ID |

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| shopName | String | 是 | 门店名称 |
| province | String | 否 | 省份 |
| city | String | 否 | 城市 |
| district | String | 否 | 区县 |
| address | String | 否 | 详细地址 |
| managerName | String | 否 | 店长姓名 |
| managerPhone | String | 否 | 店长电话 |
| meituanId | String | 否 | 美团门店ID |
| eleId | String | 否 | 饿了么门店ID |
| status | Integer | 是 | 状态：0-已停业，1-营业中，2-休息中 |

### 请求示例

```json
{
  "shopName": "门店1（更新）",
  "province": "广东省",
  "city": "深圳市",
  "district": "南山区",
  "address": "科技园路200号",
  "managerName": "李四",
  "managerPhone": "13900139000",
  "meituanId": "MT123456",
  "eleId": "ELE789",
  "status": 1
}
```

### 响应示例

```json
{
  "code": 200,
  "message": "更新成功",
  "data": null
}
```

### 业务说明

- `shopCode` 不可修改
- 状态改为 `0`（已停业）时：
  - 自动释放该门店下所有店长的门店关联
  - 店长的 `shop_id` 置为 `null`，`status` 不变（账号保留）

| 接口路径 | `DELETE /api/shops/{id}` |
| 请求格式 | Path Variable |

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| id | Long | 是 | 门店ID |

### 响应示例

```json
{
  "code": 200,
  "message": "删除成功",
  "data": null
}
```

- 执行软删除（`deleted_at` 字段标记）
- 删除后释放该门店下所有店长的门店关联
- 已有关联的评价数据保留（`shop_id` 不变）

## 6. 批量修改门店状态

### 请求信息

| 项目 | 内容 |
|------|------|
| 接口路径 | `PUT /api/shops/batch/status` |
| 请求格式 | JSON Body |

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| shopIds | Array[Long] | 是 | 门店ID列表 |
| status | Integer | 是 | 目标状态：0-已停业，1-营业中，2-休息中 |

### 请求示例

```json
{
  "shopIds": [1, 2, 3],
  "status": 0
}
```

### 响应示例

```json
{
  "code": 200,
  "message": "批量更新成功",
  "data": {
    "updatedCount": 3
  }
}
```

### 业务说明

- 批量将门店状态修改为指定状态
- 当 `status = 0`（已停业）时：
  - 自动释放这些门店下所有店长的门店关联
  - 店长的 `shop_id` 置为 `null`
- 与租户禁用联动：租户禁用时，调用此接口将旗下所有门店设为已停业

## 7. 修改营业状态

### 请求信息

| 项目 | 内容 |
|------|------|
| 接口路径 | `PUT /api/shops/{id}/status` |
| 请求格式 | Path Variable + JSON Body |

### 路径参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| id | Long | 是 | 门店ID |

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| status | Integer | 是 | 营业状态：1-营业中，2-休息中 |

### 请求示例

```json
{
  "status": 2
}
```

### 响应示例

```json
{
  "code": 200,
  "message": "状态更新成功",
  "data": null
}
```

### 业务说明

- 店长可以修改自己门店的营业状态（`shop_owner` 角色）
- 此接口仅用于快速切换营业状态，不涉及已停业

## 权限控制

| 角色 | 门店管理权限 |
|------|-------------|
| super_admin | 不可访问（仅平台侧使用） |
| tenant_admin | 完整增删改查 + 批量操作 |
| shop_owner | 仅可修改自己门店的营业状态 |

---

## 门店状态说明

| status | 名称 | 说明 |
|:------:|------|------|
| 0 | 已停业 | 门店关闭，释放店长关联 |
| 1 | 营业中 | 正常营业 |
| 2 | 休息中 | 临时休息，保留店长关联 |
