# 评价管理接口文档

## 通用接口规范

### 1) 统一返回结构

所有接口统一返回：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

- `code`：业务状态码，`200` 表示成功
- `message`：提示信息
- `data`：响应数据，列表接口通常返回分页对象，删除等操作可为 `null`

### 2) 错误码约定

| 错误码 | 含义 |
|------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未登录或登录已过期 |
| 403 | 无权限访问 |
| 404 | 资源不存在 |
| 409 | 业务冲突（如状态不允许、数据重复） |
| 500 | 服务器内部错误 |

### 3) 分页参数约定

- 请求参数统一使用：`page`（页码，从 1 开始）、`pageSize`（每页条数）
- 分页响应统一包含：`records`、`total`、`page`、`pageSize`

### 4) 时间格式约定

- 日期：`yyyy-MM-dd`
- 日期时间：`yyyy-MM-dd HH:mm:ss`
- 默认时区：`Asia/Shanghai`（东八区）

### 5) 鉴权方式约定

- 除登录/注册等白名单接口外，均需携带 JWT 访问令牌
- 请求头统一使用：`Authorization: Bearer <token>`

## 概述

- **模块说明**：评价管理
- **权限说明**：
  - 店长（`shop_owner`）：可查看和回复自己门店的评价
  - 租户管理员（`tenant_admin`）：可查看所有门店的评价，**无回复权限**
- **数据范围**：店长只能操作自己关联门店的评价

|------|------|
| 接口路径 | `GET /api/reviews` |
| 请求格式 | Query Params |

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| page | Integer | 否 | 页码，默认 1 |
| pageSize | Integer | 否 | 每页条数，默认 10 |
| platform | Integer | 否 | 平台：1-美团，2-饿了么 |

## 3. 回复评价

### 请求信息

| 项目 | 内容 |
|------|------|
| 接口路径 | `POST /api/reviews/{id}/reply` |
| 请求格式 | Path Variable + JSON Body |

### 路径参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| id | Long | 是 | 评价ID |

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| content | String | 是 | 回复内容（最多500字） |
| useTemplate | Boolean | 否 | 是否使用模板，默 false |

### 请求示例

```json
{
  "content": "非常抱歉给您带来不愉快的体验。我们已安排专人跟进处理，感谢您的反馈！",
  "useTemplate": false
}
```

### 响应示例

```json
{
  "code": 200,
  "message": "回复成功",
  "data": null
}
```

### 业务说明

- 仅 `shop_owner`（店长）可回复自己门店的评价
- 租户管理员无权回复评价
- 店长只能回复自己关联门店的评价
- 不可重复回复（已回复的评价不可修改）
- 回复后 `replyStatus` 变为 `1`，`replyTime` 记录回复时间

## 5. 评价统计

### 请求信息

| 项目 | 内容 |
|------|------|
| 接口路径 | `GET /api/reviews/statistics` |
| 请求格式 | Query Params |

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| shopId | Long | 否 | 门店ID（不传则查询所有门店汇总） |

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| code | Integer | 状态码 |
| message | String | 提示信息 |
| data | Object | 统计数据 |

#### data 结构

| 参数名 | 类型 | 说明 |
|--------|------|------|
| totalCount | Integer | 评价总数 |
| replyCount | Integer | 已回复数 |
| unReplyCount | Integer | 未回复数 |
| replyRate | Double | 回复率（百分比） |
| avgStarRating | Double | 平均评分（保留1位小数） |
| sentimentStats | Object | 情感分布 |
| platformStats | Object | 平台分布 |

#### sentimentStats 结构

| 参数名 | 类型 | 说明 |
|--------|------|------|
| positive | Integer | 好评数（4-5星） |
| neutral | Integer | 中评数（3星） |
| negative | Integer | 差评数（1-2星） |

#### platformStats 结构

| 参数名 | 类型 | 说明 |
|--------|------|------|
| meituan | Integer | 美团评价数 |
| ele | Integer | 饿了么评价数 |

### 请求示例

```http
GET /api/reviews/statistics
```

### 响应示例

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "totalCount": 100,
    "replyCount": 85,
    "unReplyCount": 15,
    "replyRate": 85.0,
    "avgStarRating": 4.2,
    "sentimentStats": {
      "positive": 60,
      "neutral": 25,
      "negative": 15
    },
    "platformStats": {
      "meituan": 65,
      "ele": 35
    }
  }
}
```

### 业务说明

- 店长查询时，默认返回自己关联门店的统计数据
- 店长可指定 `shopId` 查询单个门店（仅限自己关联的门店）
- 租户管理员可查询所有门店汇总
- 平台管理员无权限访问

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|

### 响应示例

```json

  "message": "删除成功",
  "data": null
```

- 仅可删除自定义标签（`type=custom`）
