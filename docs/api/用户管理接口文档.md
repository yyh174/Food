# 用户管理接口文档

## 通用接口规范

### 1) 统一返回结构

所有接口统一返回：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

- `code`：业务状态码，`200` 表示成功
- `message`：提示信息
- `data`：响应数据，列表接口通常返回分页对象，删除等操作可为 `null`

### 2) 错误码约定

| 错误码 | 含义 |
|------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未登录或登录已过期 |
| 403 | 无权限访问 |
| 404 | 资源不存在 |
| 409 | 业务冲突（如状态不允许、数据重复） |
| 500 | 服务器内部错误 |

### 3) 分页参数约定

- 请求参数统一使用：`page`（页码，从 1 开始）、`pageSize`（每页条数）
- 分页响应统一包含：`records`、`total`、`page`、`pageSize`

### 4) 时间格式约定

- 日期：`yyyy-MM-dd`
- 日期时间：`yyyy-MM-dd HH:mm:ss`
- 默认时区：`Asia/Shanghai`（东八区）

### 5) 鉴权方式约定

- 除登录/注册等白名单接口外，均需携带 JWT 访问令牌
- 请求头统一使用：`Authorization: Bearer <token>`

## 概述

- **模块说明**：用户管理（租户管理员使用）
- **权限说明**：仅 `tenant_admin`（租户管理员）可访问
- **数据范围**：租户管理员只能操作自己租户下的用户

## 1. 用户列表

### 请求信息

| 项目 | 内容 |
|------|------|
| 接口路径 | `GET /api/users` |
| 请求格式 | Query Params |

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| page | Integer | 否 | 页码，默认 1 |
| pageSize | Integer | 否 | 每页条数，默认 10 |
| keyword | String | 否 | 搜索关键词（匹配用户名、姓名、手机号） |
| roleCode | String | 否 | 角色筛选：tenant_admin-租户管理员，shop_owner-门店店长 |

| 参数名 | 类型 | 说明 |
|--------|------|------|
| code | Integer | 状态码 |
| message | String | 提示信息 |

| 参数名 | 类型 | 说明 |
|--------|------|------|
| records | Array | 用户列表 |
| total | Long | 总条数 |
| page | Integer | 当前页码 |
| pageSize | Integer | 每页条数 |

#### records[] 用户对象

| 参数名 | 类型 | 说明 |

## 3. 创建用户

### 请求信息

| 项目 | 内容 |
|------|------|
| 接口路径 | `POST /api/users` |
| 请求格式 | JSON Body |

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| username | String | 是 | 用户名（登录账号） |
| realName | String | 是 | 真实姓名 |
| phone | String | 否 | 手机号 |
| roleCode | String | 是 | 角色：tenant_admin-租户管理员，shop_owner-门店店长 |
| shopIds | Array[Long] | 否 | 关联门店ID列表（店长角色可指定，管理员不填） |
| password | String | 否 | 初始密码，默认 123456 |

### 请求示例

```json
{
  "username": "newuser",
  "realName": "新用户",
  "phone": "13700137000",
  "roleCode": "shop_owner",
  "shopIds": [1, 2],
  "password": "123456"
}
```

### 响应示例

```json
{
  "code": 200,
  "message": "创建成功",
  "data": {
    "id": 6,
    "username": "newuser",
    "realName": "新用户",
    "roleCode": "shop_owner",
    "status": 1,
    "createdAt": "2024-01-20 14:30:00"
  }
}
```

### 业务说明

- 用户名在同一租户下不可重复
- 初始密码默认 `123456`
- 角色说明：
  - `tenant_admin`（租户管理员）：可管理用户、门店、评价等，不绑定门店
  - `shop_owner`（门店店长）：可管理指定门店的营业状态，可绑定多个门店
- 新建用户默认状态为 `1`（正常）

## 4. 编辑用户

### 请求信息

| 项目 | 内容 |
|------|------|
| 接口路径 | `PUT /api/users/{id}` |
| 请求格式 | Path Variable + JSON Body |

### 路径参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| id | Long | 是 | 用户ID |

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| realName | String | 是 | 真实姓名 |
| phone | String | 否 | 手机号 |
| roleCode | String | 是 | 角色：tenant_admin-租户管理员，shop_owner-门店店长 |
| shopIds | Array[Long] | 否 | 关联门店ID列表 |
| status | Integer | 是 | 状态：0-禁用，1-正常 |

### 请求示例

```json
{
  "realName": "张三（更新）",
  "phone": "13900139001",
  "roleCode": "shop_owner",
  "shopIds": [1, 3],
  "status": 1
}
```

### 响应示例

```json
{
  "code": 200,
  "message": "更新成功",
  "data": null
}
```

### 业务说明

- `username` 不可修改
- 状态改为 `0`（禁用）时：
  - 用户无法登录
  - 自动释放该用户的所有门店关联（`shop_ids` 清空）
- `shopIds` 更新时会替换原有的门店关联

### 请求信息

| 项目 | 内容 |
|------|------|
| 接口路径 | `POST /api/users/{id}/password/reset` |
| 请求格式 | Path Variable |

### 路径参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| id | Long | 是 | 用户ID |

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| newPassword | String | 否 | 新密码，默认 123456 |

### 请求示例

```json
{
  "newPassword": "123456"
}
```

### 响应示例

```json
{
  "code": 200,
  "message": "密码重置成功",
  "data": {
    "newPassword": "123456"
  }
}
```

### 业务说明

- 租户管理员可为租户下的任何用户重置密码
- 默认重置为 `123456`
- 重置后可返回新密码供管理员告知用户

## 8. 修改门店关联（批量）

### 请求信息

| 项目 | 内容 |
|------|------|
| 接口路径 | `PUT /api/users/{id}/shops` |
| 请求格式 | Path Variable + JSON Body |

### 路径参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| id | Long | 是 | 用户ID |

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| shopIds | Array[Long] | 是 | 新的关联门店ID列表（空数组表示解除所有关联） |

### 请求示例

```json
{
  "shopIds": [1, 2, 3]
}
```

### 响应示例

```json
{
  "code": 200,
  "message": "门店关联已更新",
  "data": null
}
```

### 业务说明

- 批量替换用户的门店关联
- 传入空数组可清除所有关联
- 仅对 `shop_owner`（店长）角色有效
- `tenant_admin`（租户管理员）不可指定门店关联

## 权限控制

| 角色 | 用户管理权限 |
|------|-------------|
| super_admin | 不可访问（仅平台侧使用） |
| tenant_admin | 完整增删改查 + 重置密码 + 修改门店关联 |
| shop_owner | 不可访问 |

---

## 角色说明

| roleCode | roleName | 门店关联 | 说明 |
|----------|----------|:--------:|------|
| tenant_admin | 租户管理员 | 无 | 可管理租户下所有资源 |
| shop_owner | 门店店长 | 可多个 | 可管理指定门店的营业状态 |

---

## 用户状态说明

| status | 名称 | 说明 |
|:------:|------|------|
| 0 | 禁用 | 无法登录，数据保留 |
| 1 | 正常 | 正常使用 |

---

## 禁用用户时的联动操作

| 关联项 | 处理方式 |
|--------|----------|
| 门店关联 | 自动解除（`shop_ids` 清空） |
| 登录状态 | 强制登出 |
| 历史数据 | 保留不变 |
