# 租户管理接口文档

## 通用接口规范

### 1) 统一返回结构

所有接口统一返回：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

- `code`：业务状态码，`200` 表示成功
- `message`：提示信息
- `data`：响应数据，列表接口通常返回分页对象，删除等操作可为 `null`

### 2) 错误码约定

| 错误码 | 含义 |
|------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未登录或登录已过期 |
| 403 | 无权限访问 |
| 404 | 资源不存在 |
| 409 | 业务冲突（如状态不允许、数据重复） |
| 500 | 服务器内部错误 |

### 3) 分页参数约定

- 请求参数统一使用：`page`（页码，从 1 开始）、`pageSize`（每页条数）
- 分页响应统一包含：`records`、`total`、`page`、`pageSize`

### 4) 时间格式约定

- 日期：`yyyy-MM-dd`
- 日期时间：`yyyy-MM-dd HH:mm:ss`
- 默认时区：`Asia/Shanghai`（东八区）

### 5) 鉴权方式约定

- 除登录/注册等白名单接口外，均需携带 JWT 访问令牌
- 请求头统一使用：`Authorization: Bearer <token>`

## 概述

- **模块说明**：租户管理（平台管理员使用）
- **权限说明**：仅 `super_admin`（平台管理员）可访问
- **数据范围**：平台管理员可操作所有租户数据

## 1. 租户列表

### 请求信息

| 项目 | 内容 |
|------|------|
| 接口路径 | `GET /api/tenants` |
| 请求格式 | Query Params |

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| page | Integer | 否 | 页码，默认 1 |
| pageSize | Integer | 否 | 每页条数，默认 10 |
| keyword | String | 否 | 搜索关键词（匹配租户编码、名称） |
| status | Integer | 否 | 状态筛选：0-禁用，1-正常 |

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| code | Integer | 状态码 |
| message | String | 提示信息 |
| data | Object | 数据对象 |

#### data 结构

| 参数名 | 类型 | 说明 |
|--------|------|------|
| records | Array | 租户列表 |
| total | Long | 总条数 |
| page | Integer | 当前页码 |
| pageSize | Integer | 每页条数 |

#### records[] 租户对象

| 参数名 | 类型 | 说明 |
|--------|------|------|
| id | Long | 租户ID |
| tenantCode | String | 租户编码 |
| tenantName | String | 租户名称（品牌名称） |
| logoUrl | String | 品牌Logo |
| status | Integer | 状态：0-禁用，1-正常 |
| contactName | String | 联系人 |
| contactPhone | String | 联系电话 |
| inviteCode | String | 员工邀请码 |
| apiQuota | Integer | API配额 |
| apiUsed | Integer | 已使用API次数 |
| expireTime | String | 到期时间 |
| shopCount | Integer | 门店数量 |
| userCount | Integer | 用户数量 |
| createdAt | String | 创建时间 |


## 3. 创建租户

### 请求信息

| 项目 | 内容 |
|------|------|
| 接口路径 | `POST /api/tenants` |
| 请求格式 | JSON Body |

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| tenantName | String | 是 | 租户名称（品牌名称） |
| logoUrl | String | 否 | 品牌Logo |
| contactName | String | 否 | 联系人 |
| contactPhone | String | 否 | 联系电话 |
| apiQuota | Integer | 否 | API配额，默认 1000 |
| expireTime | String | 否 | 到期时间 |

### 请求示例

```json
{
  "tenantName": "新品牌",
  "logoUrl": "https://example.com/new-logo.png",
  "contactName": "李四",
  "contactPhone": "13900139000",
  "apiQuota": 1000,
  "expireTime": "2025-12-31 23:59:59"
}
```

### 响应示例

```json
{
  "code": 200,
  "message": "创建成功",
  "data": {
    "id": 4,
    "tenantCode": "TN04",
    "tenantName": "新品牌",
    "inviteCode": "TXXXXXX01",
    "status": 1,
    "createdAt": "2024-01-20 14:30:00"
  }
}
```

### 业务说明

- `tenantCode` 由系统自动生成，格式：`TN` + 2位序号（如 TN01）
- `inviteCode` 由系统自动生成，用于员工注册时加入租户
- 新建租户默认状态为 `1`（正常）
- 同时为租户创建一个默认管理员账号（`{tenantCode}_admin`）

## 4. 编辑租户

### 请求信息

| 项目 | 内容 |
|------|------|
| 接口路径 | `PUT /api/tenants/{id}` |
| 请求格式 | Path Variable + JSON Body |

### 路径参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| id | Long | 是 | 租户ID |

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| tenantName | String | 是 | 租户名称（品牌名称） |
| logoUrl | String | 否 | 品牌Logo |
| contactName | String | 否 | 联系人 |
| contactPhone | String | 否 | 联系电话 |
| apiQuota | Integer | 否 | API配额 |
| expireTime | String | 否 | 到期时间 |
| status | Integer | 是 | 状态：0-禁用，1-正常 |

### 请求示例

```json
{
  "tenantName": "品牌1（更新）",
  "logoUrl": "https://example.com/updated-logo.png",
  "contactName": "王五",
  "contactPhone": "13700137000",
  "apiQuota": 2000,
  "expireTime": "2026-12-31 23:59:59",
  "status": 1
}
```

### 响应示例

```json
{
  "code": 200,
  "message": "更新成功",
  "data": null
}
```

### 业务说明

- `tenantCode` 不可修改
- 状态改为 `0`（禁用）时：
  - 该租户下所有用户账号被禁用（`status = 0`）
  - 该租户下所有门店状态设为 `2`（休息中）
  - 用户被禁用但数据保留，只是不能登录

## 5. 删除租户

### 请求信息

| 项目 | 内容 |
|------|------|
| 接口路径 | `DELETE /api/tenants/{id}` |
| 请求格式 | Path Variable |

### 路径参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| id | Long | 是 | 租户ID |

### 响应示例

```json
{
  "code": 200,
  "message": "删除成功",
  "data": null
}
```

### 业务说明

- 执行软删除（`deleted_at` 字段标记）
- 删除后该租户下所有数据保留（门店、用户、评价、工单等）
- 用户无法再登录访问

## 权限控制

| 角色 | 租户管理权限 |
|------|-------------|
| super_admin | 完整增删改查 + 重置邀请码 + 查看关联数据 |
| tenant_admin | 不可访问（仅平台侧使用） |
| shop_owner | 不可访问 |

---

## 租户状态说明

| status | 名称 | 说明 |
|:------:|------|------|
| 0 | 禁用 | 用户无法登录，门店休息中，数据保留 |
| 1 | 正常 | 正常使用 |

---

## 租户禁用时的联动操作

| 关联项 | 处理方式 |
|--------|----------|
| 用户 | `status = 0`（禁用），数据保留，无法登录 |
| 门店 | `status = 2`（休息中），数据保留 |
| 评价数据 | 保留不变 |
| 工单数据 | 保留不变 |
| 回复模板 | 保留不变 |
