# 数据看板接口文档

## 通用接口规范

### 1) 统一返回结构

所有接口统一返回：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

- `code`：业务状态码，`200` 表示成功
- `message`：提示信息
- `data`：响应数据，列表接口通常返回分页对象，删除等操作可为 `null`

### 2) 错误码约定

| 错误码 | 含义 |
|------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未登录或登录已过期 |
| 403 | 无权限访问 |
| 404 | 资源不存在 |
| 409 | 业务冲突（如状态不允许、数据重复） |
| 500 | 服务器内部错误 |

### 3) 分页参数约定

- 请求参数统一使用：`page`（页码，从 1 开始）、`pageSize`（每页条数）
- 分页响应统一包含：`records`、`total`、`page`、`pageSize`

### 4) 时间格式约定

- 日期：`yyyy-MM-dd`
- 日期时间：`yyyy-MM-dd HH:mm:ss`
- 默认时区：`Asia/Shanghai`（东八区）

### 5) 鉴权方式约定

- 除登录/注册等白名单接口外，均需携带 JWT 访问令牌
- 请求头统一使用：`Authorization: Bearer <token>`

## 概述

- **模块说明**：数据看板
- **权限说明**：
  - 租户管理员（`tenant_admin`）：查看所有门店汇总数据
  - 店长（`shop_owner`）：查看自己门店的数据
- **数据范围**：店长只能查看自己关联门店的数据

## 1. 看板首页统计

### 请求信息

| 项目 | 内容 |
|------|------|
| 接口路径 | `GET /api/dashboard/summary` |
| 请求格式 | Query Params |

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| date | String | 否 | 查询日期（格式：yyyy-MM-dd），默认今天 |
| shopId | Long | 否 | 门店ID（店长必填，租户管理员可选） |

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| code | Integer | 状态码 |
| message | String | 提示信息 |
| data | Object | 统计数据 |

#### data 结构

| 参数名 | 类型 | 说明 |
|--------|------|------|
| date | String | 查询日期 |
| totalOrders | Integer | 总订单量 |
| totalAmount | Decimal | 总营业额（元） |
| totalReviews | Integer | 总评价数 |
| negativeReviews | Integer | 差评数（1-2星） |
| avgRating | Decimal | 平均评分 |
| meituan | Object | 美团数据 |
| ele | Object | 饿了么数据 |

#### meituan / ele 结构

| 参数名 | 类型 | 说明 |
|--------|------|------|
| orders | Integer | 订单量 |
| amount | Decimal | 营业额（元） |
| reviews | Integer | 评价数 |
| negativeReviews | Integer | 差评数 |
| avgRating | Decimal | 平均评分 |

### 请求示例

```http
GET /api/dashboard/summary?date=2024-01-15
```

### 响应示例

## 2. 同比环比数据

### 请求信息

| 项目 | 内容 |
|------|------|
| 接口路径 | `GET /api/dashboard/trend` |
| 请求格式 | Query Params |

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| date | String | 否 | 查询日期（格式：yyyy-MM-dd），默认今天 |
| shopId | Long | 否 | 门店ID（店长必填，租户管理员可选） |

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| code | Integer | 状态码 |
| message | String | 提示信息 |
| data | Object | 同比环比数据 |

#### data 结构

| 参数名 | 类型 | 说明 |
|--------|------|------|
| current | Object | 当日数据 |
| yesterday | Object | 昨日数据 |
| lastWeek | Object | 上周同天数据 |
| lastMonth | Object | 上月同天数据 |
| trend | Object | 趋势数据 |

#### current / yesterday / lastWeek / lastMonth 结构

| 参数名 | 类型 | 说明 |
|--------|------|------|
| date | String | 日期 |
| orders | Integer | 订单量 |
| amount | Decimal | 营业额 |
| reviews | Integer | 评价数 |
| avgRating | Decimal | 平均评分 |

#### trend 结构

| 参数名 | 类型 | 说明 |
|--------|------|------|
| ordersVsYesterday | Decimal | 订单量较昨日变化百分比 |
| amountVsYesterday | Decimal | 营业额较昨日变化百分比 |
| ordersVsLastWeek | Decimal | 订单量较上周变化百分比 |
| amountVsLastWeek | Decimal | 营业额较上周变化百分比 |
| ordersVsLastMonth | Decimal | 订单量较上月变化百分比 |
| amountVsLastMonth | Decimal | 营业额较上月变化百分比 |

### 请求示例

```http
GET /api/dashboard/trend?date=2024-01-15
```

### 响应示例

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "current": {
      "date": "2024-01-15",
      "orders": 150,
      "amount": 22500.00,
      "reviews": 45,
      "avgRating": 4.5
    },
    "yesterday": {
      "date": "2024-01-14",
      "orders": 140,
      "amount": 21000.00,
      "reviews": 40,
      "avgRating": 4.4
    },
    "lastWeek": {
      "date": "2024-01-08",
      "orders": 130,
      "amount": 19500.00,
      "reviews": 38,
      "avgRating": 4.3
    },
    "lastMonth": {
      "date": "2023-12-15",
      "orders": 120,
      "amount": 18000.00,
      "reviews": 35,
      "avgRating": 4.2
    },
    "trend": {
      "ordersVsYesterday": 7.14,
      "amountVsYesterday": 7.14,
      "ordersVsLastWeek": 15.38,
      "amountVsLastWeek": 15.38,
      "ordersVsLastMonth": 25.00,
      "amountVsLastMonth": 25.00
    }
  }
}
```

### 说明

- 变化百分比计算公式：`((当前值 - 对比值) / 对比值) * 100`
- 正数表示增长，负数表示下降

## 3. 趋势图表数据

### 请求信息

| 项目 | 内容 |
|------|------|
| 接口路径 | `GET /api/dashboard/chart` |
| 请求格式 | Query Params |

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| startDate | String | 是 | 开始日期（格式：yyyy-MM-dd） |
| endDate | String | 是 | 结束日期（格式：yyyy-MM-dd） |
| metric | String | 否 | 指标：orders-订单量，amount-营业额，reviews-评价数，avgRating-平均评分，默认 orders |
| shopId | Long | 否 | 门店ID（店长必填，租户管理员可选） |

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| code | Integer | 状态码 |
| message | String | 提示信息 |
| data | Object | 图表数据 |

#### data 结构

| 参数名 | 类型 | 说明 |
|--------|------|------|
| metric | String | 指标类型 |
| labels | Array[String] | X轴标签（日期列表） |
| series | Array | 数据系列 |

#### series[] 结构

| 参数名 | 类型 | 说明 |
|--------|------|------|
| name | String | 系列名称：total/meituan/ele |
| data | Array[Number] | 数据值列表 |

### 请求示例

```http
GET /api/dashboard/chart?startDate=2024-01-01&endDate=2024-01-15&metric=orders
```

### 响应示例

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "metric": "orders",
    "labels": ["01-01", "01-02", "01-03", "01-04", "01-05", "01-06", "01-07"],
    "series": [
      { "name": "total", "data": [150, 145, 160, 155, 170, 165, 180] },
      { "name": "meituan", "data": [100, 95, 105, 100, 110, 105, 120] },
      { "name": "ele", "data": [50, 50, 55, 55, 60, 60, 60] }
    ]
  }
}
```

## 4. 门店排名

### 请求信息

| 项目 | 内容 |
|------|------|
| 接口路径 | `GET /api/dashboard/ranking` |
| 请求格式 | Query Params |

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| date | String | 否 | 查询日期（格式：yyyy-MM-dd），默认今天 |
| metric | String | 否 | 排名指标：orders-订单量，amount-营业额，avgRating-平均评分，默认 orders |
| order | String | 否 | 排序：asc-升序，desc-降序，默认 desc |
| page | Integer | 否 | 页码，默认 1 |
| pageSize | Integer | 否 | 每页条数，默认 10 |

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| code | Integer | 状态码 |
| message | String | 提示信息 |
| data | Object | 排名数据 |

#### data 结构

| 参数名 | 类型 | 说明 |
|--------|------|------|
| records | Array | 排名列表 |
| total | Long | 总条数 |
| page | Integer | 当前页码 |
| pageSize | Integer | 每页条数 |

#### records[] 结构

| 参数名 | 类型 | 说明 |
|--------|------|------|
| rank | Integer | 排名 |
| shopId | Long | 门店ID |
| shopName | String | 门店名称 |
| orders | Integer | 订单量 |
| amount | Decimal | 营业额 |
| avgRating | Decimal | 平均评分 |
| value | Number | 排名指标值（订单量/营业额/平均评分） |

### 请求示例

```http
GET /api/dashboard/ranking?date=2024-01-15&metric=amount&order=desc
```

### 响应示例

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "records": [
      { "rank": 1, "shopId": 1, "shopName": "门店1", "orders": 100, "amount": 15000.00, "avgRating": 4.6, "value": 15000.00 },
      { "rank": 2, "shopId": 2, "shopName": "门店2", "orders": 80, "amount": 12000.00, "avgRating": 4.5, "value": 12000.00 },
      { "rank": 3, "shopId": 3, "shopName": "门店3", "orders": 60, "amount": 9000.00, "avgRating": 4.4, "value": 9000.00 }
    ],
```
